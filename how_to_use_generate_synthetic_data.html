Using the Synthetic Data Generator Script

Overview

The synthetic data generator script creates multi-modal medical scenario data including text, images, audio, video, and physiological data for both emergency and non-emergency scenarios.

Prerequisites

  • Python 3.x installed
  • Required packages: 
    pip install numpy Pillow opencv-python pyttsx3 tqdm

Script Configuration

The script can be configured by modifying these key variables:

  • NUM_INSTANCES_PER_SCENARIO: Number of samples to generate per scenario (default: 1000)
  • SCENARIOS: List of scenarios with their details:
    • Emergency scenarios (id: 1-12)
    • Non-emergency scenarios (id: 13-20)

Generated Data Structure

The script creates the following directory structure:

├── text_data/           # Text descriptions of scenarios
├── image_data/          # Generated facial expression images
├── audio_data/          # Synthesized voice recordings
├── video_data/          # Simple animation videos
├── physiological_data/  # Simulated vital signs in JSON format
└── metadata/            # SQLite database with sample metadata

Running the Script

  1. Save the script as generate_synthetic_data.py
  2. Open a terminal/command prompt
  3. Navigate to the script's directory
  4. Run the script: 
    python generate_synthetic_data.py

Customizing Scenarios

To add or modify scenarios, edit the SCENARIOS list in the script:

SCENARIOS = [
    {
        'id': new_id,
        'name': 'Scenario Name',
        'description': 'Scenario Description',
        'emergency': 'Yes/No'
    },
    # Add more scenarios...
]

Output Data

For each scenario instance, the script generates:

  • Text files containing scenario descriptions
  • PNG images showing facial expressions
  • WAV audio files with synthesized speech
  • MP4 video files with simple animations
  • JSON files containing physiological data
  • SQLite database entries with metadata

Performance Notes

  • The script uses multiprocessing to generate data in parallel
  • Progress is displayed using a progress bar
  • Generation time depends on the number of instances and available CPU cores

Error Handling

The script includes:

  • Automatic directory creation
  • Warning suppression for cleaner output
  • Database connection management
